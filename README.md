# urbanFlowGen
urbanFlowGen is an open-source Python library for generating synthetic urban Computational Fluid Dynamics (CFD) scenarios tailored for machine learning applications. It provides a fully automated pipeline for:

- Step 1: Generating randomized or real-city 3D urban geometries with customizable features,
- Step 2: Creating property files for generating computational meshes using open-source grid generation tools,
- Step 3: Creating property files for setting up CFD simulations for both CPU- and GPU-based solvers.

The library is modular and extensible, making it easy to integrate with custom solvers or simulation workflows. It is designed to support researchers and practitioners working on data-driven urban flow modeling, offering a scalable solution for producing diverse and physically realistic training datasets.

When executing the bash script "virtualEnv.sh", a Python virtual environment is created, the modules listed in "requirements.txt" are installed with pip, and the virtual environment is activated. The directory of the virtual environment can be adjusted in "VENV_DIR=...".

All user-specific parameters can be set in the file "config.toml". For real-city domains, the "config_realCity.toml" needs to be customized in addition to "config.toml". Details about each parameter will be published soon in a journal publication.

Step 1, generating geometries of urban CFD scenarios, can be realized with "geometryGenerator.py" or "geometryGenerator_realCity.py". There are the following two functionalities:

I. Synthetic data: "geometryGenerator.py"
Users can define key geometric properties—including domain dimensions, building types, and placement rules—using a set of normalized parameters. The generator supports rectangular, circular, and tower-like buildings with customizable sizes, aspect ratios, and placement logic, including spacing constraints for simulation stability. Buildings are randomly distributed within a defined urban section, with optional rotation to simulate varying wind angles. Additionally, non-geometrical roads can be added to enhance realism by reflecting typical urban layouts without affecting the exported geometry. When executing "geometryGenerator.py", a geometry ID needs to be inserted, e.g., "geometryGenerator.py 1" if the geometry should have ID=1. The ID and all properties are stored in a JavaScript Object Notation (JSON) file once a geometry is generated. Per default, the STL and JSON files are stored in "./ID/stl/".

II. Real-world data: "geometryGenerator_realCity.py"
The framework enables the extraction and processing of realistic urban geometries from either the Overture Maps [1] or the Global Building Atlas provided by the Technical University of Munich (TUM)[2]. For this purpose, urbanFlowGen is coupled with the corresponding Overture or TUM library to retrieve real-city building footprints and generate three-dimensional domain representations suitable for flow simulations. 

The two approaches serve complementary objectives: the synthetic generator allows controlled and systematic variation of urban parameters for structured dataset creation, whereas the integration of real-city geometries introduces authentic morphological variability to enhance realism and model generalizability. In both modes, the resulting 3D models are exported in stereolithography (STL) format, ensuring compatibility with most state-of-the-art CFD solvers.

Steps 2 and 3 are demonstrated for a use cases with the open source simulation framework m-AIA [3]. For both, grid generation and running the simulation, the property file "geometry.toml" is required and generated by urbanFlowGen. Here, the number of segments and the boundary condition for each segment is defined. This file is generated at the end of Step 1.

Step 2, generating property files for grid generation, executes the file "gridGenerator.py". The parameters include user-defined global and local grid refinement. Further information about the m-AIA grid generator can be found in [4].

Step 3, generating property files for simulating, uses the cumulant lattice Boltzmann (CLB) module of m-AIA to simulate high-Reynolds-number urban airflow. The CLB method offers enhanced stability and accuracy over traditional BGK or MRT schemes by relaxing cumulants instead of moments, making it ideal for capturing turbulence, vortex shedding, and complex near-wall dynamics in city-scale CFD. Simulations are configured via the "properties_run.toml" property file generated using urbanFlowGen, which includes settings for time-stepping, ramped Reynolds number initialization, restart control, and time-averaged output. Optional monitoring of flow quantities at inlet/outlet can aid early-stage validation. Further details about the CLB method are provided in [5].

UrbanFlowGen is executed with the bash script "urbanFlowGen.sh". The user needs to comment in one of the two geometry generator files. After this, the script automates the generation and submission of multiple urban flow simulation cases. For each case within a user-defined integer range (case IDs), the script:

    - Activates a Python virtual environment.
    - Creates a parent directory named by the case ID.
    - Creates necessary directories for geometry (stl/) and output (out/).
    - Generates geometry, grid, and simulation property files using Python scripts.
    - Organizes all files into a case-specific folder (e.g., ./42/).
    - Copies and customizes SLURM job scripts, adjusting the job name per case.
    - Submits the jobs to the SLURM scheduler:

        First, the grid generation job.
        Then, the simulation job, which starts only after the grid job finishes (--dependency=afterok).

This enables scalable and reproducible generation of parametric CFD cases for urban flow studies.

The default slurm job scripts for grid generation ("grid.sh") and simulation ("run_gpu.sh") with m-AIA are compatible with the NVIDIA A-100 GPU partition on the JURECA-DC supercomputer at the Jülich Supercomputing Centre (JSC), Forschungszentrum Jülich GmbH, Germany [6].
